## Architecture Recommendation

For a fast-growing food delivery startup collecting GPS location logs, customer text reviews, payment transactions, and restaurant menu images, the most suitable storage architecture is a Data Lakehouse.

A Data Lakehouse combines the flexibility of a data lake with the structured querying capabilities of a data warehouse. This makes it ideal for handling diverse and large-scale data generated by modern digital platforms.

First, the startup collects multiple data types such as structured data (payment transactions), semi-structured data (GPS logs and JSON records), and unstructured data (customer reviews and menu images). A traditional data warehouse is optimized mainly for structured data and would struggle to efficiently store unstructured data such as images or text reviews. A data lakehouse allows storing all these formats in a unified storage system while still enabling analytical queries.

Second, scalability is important for a fast-growing startup. A lakehouse architecture built on cloud object storage can scale easily as the amount of delivery logs, images, and customer interactions grows. This allows the company to store large volumes of historical data without significantly increasing infrastructure costs.

Third, a lakehouse supports both analytical processing and machine learning workloads. The startup can analyze delivery performance, customer behavior, and payment patterns while also training machine learning models for tasks such as delivery time prediction, restaurant recommendations, and sentiment analysis of reviews.

Therefore, a Data Lakehouse architecture provides the flexibility, scalability, and analytics capabilities required for a modern food delivery platform.
